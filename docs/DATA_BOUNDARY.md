# 数据与能力边界

## 对外可见内容

本仓库的所有案例、编号、问题描述、时间、数量、责任角色和界面状态均为合成示例。它们的作用是说明信息如何被归一、分诊、派发、复盘，不代表新石器或任何企业的真实运营记录。

## 需要企业侧确认的内容

- 可接入的数据字段、脱敏规则与保留期限；
- 现有工单、客服、运营与技术团队之间的责任边界；
- P0/P1/P2/P3 的实际响应规则；
- 人工审批节点、升级路径和结案标准；
- 评价指标的计算口径。

## 不在方案范围内的能力

- 不控制车辆、门锁、路线、充电或调度；
- 不替代既有查车、发车、开门、充电调度和经营问答能力；
- 不自动认定事故责任、赔偿、恢复时间或最终技术根因；
- 不将模型输出作为无需审核的运营决定。

## 上线前条件

只有在获得赛事指定飞书权限、完成字段盘点和权限验收后，才进入单 Base 数据模型、AI 节点和工作流的实施与验收阶段。本公开演示不表示已获得上述权限或已进行真实数据接入。
